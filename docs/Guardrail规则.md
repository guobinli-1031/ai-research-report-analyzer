# Guardrail规则文档

## 规则概述

6条合规规则，每条为独立检查函数，由`run_all()`串联执行。每个函数返回 `(处理后数据, 违规列表)`。

## 规则详细定义

### G1 — 投资建议检查
**触发关键词**: 建议买入|建议卖出|推荐买入|推荐卖出|强烈推荐|建议持有|建议加仓|建议减仓|值得投资|应该买入|应该卖出

**替换映射**:
- 建议买入 → 研报评级为买入
- 建议卖出 → 研报评级为卖出
- 推荐买入 → 研报评级为买入
- 推荐卖出 → 研报评级为卖出
- 强烈推荐 → 研报给予推荐评级
- 建议持有 → 研报评级为持有
- 建议加仓 → 研报评级为增持
- 建议减仓 → 研报评级为减持
- 值得投资 → 研报认为具有投资价值
- 应该买入 → 研报评级为买入
- 应该卖出 → 研报评级为卖出

### G2 — 编造数据检查
检查JSON中各字段值：null → "未披露"，"" → "未披露"，[] → "未披露"

### G3 — 外部引用检查
**匹配模式**: Wind数据、同花顺、东方财富、Choice、Bloomberg、据*统计、根据*数据
**处理**: 包含匹配关键词的整句删除，替换为"[已移除外部引用]"

### G4 — 来源标注检查
**正则**: `\(p\.\d+\)`
**处理**: 含实质性数字但无来源标注的句子记录违规（不修改文本）

### G5 — AI推断标注检查
检查`ai_supplement`数组中每项的`risk`字段是否以`[AI推断]`开头

### G6 — 免责声明检查
如末尾无"本报告由AI自动生成"，追加完整免责声明：
```
本报告由AI自动生成，仅供参考，不构成任何投资建议。
投资有风险，入市需谨慎。AI分析结果可能存在偏差或遗漏，
请以原始研报及官方披露信息为准。
```

## 测试用例（GT1-GT8）

### GT1: G1单关键词替换
输入: `"我们建议买入该股票"`
期望: 结果不含"建议买入"，含"研报评级为买入"，1条违规

### GT2: G1多关键词替换
输入: `"强烈推荐该标的，建议加仓至重仓"`
期望: 2条违规，两处均已替换

### GT3: G2 null值替换
输入: `{"revenue": null, "profit": "", "name": "ABC"}`
期望: revenue→"未披露", profit→"未披露", name保持"ABC"

### GT4: G2 空字符串替换
输入: `{"eps": "", "pe": 15}`
期望: eps→"未披露"

### GT5: G3 Wind数据移除
输入: `"根据Wind数据，该公司营收增长15%。行业前景良好。"`
期望: 不含"Wind数据"，含"[已移除外部引用]"，"行业前景良好"保留

### GT6: G5 缺少[AI推断]标注
输入: `{"ai_supplement": [{"risk": "行业政策风险", "rationale": "..."}]}`
期望: 1条违规，"缺少[AI推断]"

### GT7: G5 已有[AI推断]标注
输入: `{"ai_supplement": [{"risk": "[AI推断] 宏观环境可能影响出口"}]}`
期望: 0条违规

### GT8: G6 缺少免责声明
输入: `"这是一份完整的分析报告。"`
期望: 结果含"本报告由AI自动生成"，1条违规

## 函数接口规格

```python
def check_g1(text: str) -> tuple[str, list[str]]   # (处理后文本, 违规列表)
def check_g2(data: Any) -> tuple[Any, list[str]]    # (处理后数据, 违规列表)
def check_g3(text: str) -> tuple[str, list[str]]    # (处理后文本, 违规列表)
def check_g4(text: str) -> tuple[str, list[str]]    # (原始文本, 违规列表)
def check_g5(data: Any) -> tuple[Any, list[str]]    # (原始数据, 违规列表)
def check_g6(text: str) -> tuple[str, list[str]]    # (处理后文本, 违规列表)
def run_all(text: str) -> dict                      # {"cleaned_text": str, "violations": list}
```
