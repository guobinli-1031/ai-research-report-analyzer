# AI研报解读助手 (RRA) — 产品需求文档 (PRD)

> 文档版本：v1.0
> 创建时间：2026-06-18

## 1. 产品概述

AI研报解读助手（Research Report Analyzer，简称RRA）是一个基于大语言模型的券商研报结构化分析工具。输入PDF格式的券商研报，通过DeepSeek API进行四维结构化分析，输出包含要素脱水、逻辑链解构、多源对比、风险透视的完整Markdown报告。

## 2. 目标用户

- 个人投资者：快速理解研报核心观点，识别潜在偏差
- 买方分析师：对比多份研报，发现逻辑漏洞
- 风控人员：透视研报中未充分披露的风险

## 3. 核心功能

### 3.1 范式1 — 要素脱水（Extraction）
从冗长研报中提取N个关键观点，标注出处和置信度。

### 3.2 范式2 — 逻辑链解构（Logic）
Multi-Agent模式：分别分析核心论点、证据链、逻辑漏洞，再整合为结构化报告。

### 3.3 范式3 — 多源对比（Comparison）
对比两份以上研报在同一标的上的观点差异，发现共识和分歧。

### 3.4 范式4 — 风险透视（Risk）
三层风险分析：明示风险、隐含风险、AI补充风险（需标注[AI推断]）。

## 4. 合规引擎（Guardrail）

6条规则确保输出合规性：

| 规则 | 名称 | 功能 |
|------|------|------|
| G1 | 投资建议检查 | 替换直接建议措辞 |
| G2 | 编造数据检查 | 空值填充"未披露" |
| G3 | 外部引用检查 | 移除Wind/同花顺等引用 |
| G4 | 来源标注检查 | 数据未标(p.X)时告警 |
| G5 | AI推断标注 | 强制[AI推断]前缀 |
| G6 | 免责声明 | 缺失时自动追加 |

## 5. 技术架构

```
CLI → pdf_parser.py → llm_client.py → prompts.py → guardrail.py → Markdown Report
```

- Python 3.10+
- PyMuPDF（PDF解析）
- DeepSeek API（LLM，兼容OpenAI SDK）
- pytest（测试）

## 6. 非功能需求

- 60秒API超时，失败重试1次
- 支持单份和多份PDF输入
- 输出Markdown格式，支持stdout和文件输出
